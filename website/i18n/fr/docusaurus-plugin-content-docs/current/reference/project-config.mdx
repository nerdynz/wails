# Configuration du projet

La configuration du projet se trouve dans le fichier `wails.json` du répertoire du projet. La structure de la configuration est :

```json5
{
  // Version de la config du projet
  "version": "",
  // Nom du projet
  "name": "",
  // Chemin relatif vers le dosier contenant les assets compilés. Normalement, cette information est déduite et ce champ peu rester vide.
  "assetdir": "",
  // Liste de dossiers (séparés par des virgules) à déclancher lors de rechargements. Seulement utilisé dans des configurations avancées. 
  "reloaddirs": "",
  // Le dossier où les fichiers construits sont stockés. Défaut : "build"
  "build:dir": "",
  // Chemin relatif vers le dossier contenant la partie frontend. Défaut : "frontend"
  "frontend:dir": "",
  // Commande pour installer les dépendances nodes depuis le dossier contenant la partie frontend. - Généralement `npm install`
  "frontend:install": "",
  // Commande pour construire les ressources du frontend depuis le dossier contenant cette partie. - Généralement `npm run build`
  "frontend:build": "",
  // Commande remplacée par "frontend:dev:build". Si "frontend:dev:build" n'est pas défini, ce paramètre sera alors utilisé. Si elle n'est pas définie, "frontend:build" sera utilisé à la place.
  "frontend:dev": "",
  // Commande équivalente à "frontend:build". Si non définie, la valeur de "frontend:dev" sera alors utilisée.
  "frontend:dev:build": "",
  // Commande équivalente à "frontend:install". Si non définie, la valeur de "frontend:install" sera alors utilisée.
  "frontend:dev:install": "",
  // Cette commande est exécutée dans un processus séparée lorsque `wails dev` est appeelé. Utile pour lancer des outils tiers de débuggage ou des serveurs de développement tiers.
  "frontend:dev:watcher": "",
  // URL vers un serveur de développement tiers pouvant être utilisé pour exposer les ressources. Par exemple : Vite. Si la valeur définir est "auto", l'url sera alors déduite des logs de Vite.
  "frontend:dev:serverUrl": "",
  // Chemin relatif vers le dossier qui contiendra les modules JS auto-générés
  "wailsjsdir": "",
  // Nom du binaire
  "outputfilename": "",
  // Temps d'attente par défaut avant que le serveur de développement effectue un rechargement après modification d'une ressource.
  "debounceMS": 100,
  // Adresse à attacher au serveur de développement de wails. Défaut: localhost:34115
  "devServer": "",
  // Arguments à passer à l'application dans le shell lorsque l'application est en "mode dev"
  "appargs": "",
  // Définit si les hooks de build doivent être exécutés bien qu'ils soient définis pour un système d'exploitation autre que le système d'exploitation hôte.
  "runNonNativeBuildHooks": false,
  "preBuildHooks": {
    // La commande qui sera exécutée avant un build du GOOS/GOARCH spécifié : ${platform} est remplacée par "GOOS/GOARCH". Le hook "GOOS/GOARCH" est exécuté avant les hooks "GOOS/*" et "*/*".
    "GOOS/GOARCH": "",
    // La commande qui sera exécutée avant une build du GOOS spécifié : ${platform} est remplacée par "GOOS/GOARCH". Le hook "GOOS/*" est exécuté avant le hook "*/*".
    "GOOS/*": "",
    // La commande qui sera exécutée avant chaque build: ${platform} est remplacé par "GOOS/GOARCH".
    "*/*": ""
  },
  "postBuildHooks": {
    // La ommande qui sera exécutée après un build du GOOS/GOARCH spécifié : ${platform} est remplacé par "GOOS/GOARCH" et ${bin} par le chemin d'accès au binaire compilé. Le hook "GOOS/GOARCH" est exécuté avant les hooks "GOOS/*" et "*/*".
    "GOOS/GOARCH": "",
    // La commande qui sera exécutée après un build du GOOS spécifié : ${platform} est remplacé par "GOOS/GOARCH" et ${bin} par le chemin d'accès au binaire compilé. Le hook "GOOS/*" est exécuté avant le hook "*/*".
    "GOOS/*": "",
    // La commande qui sera exécutée après chaque build : ${platform} est remplacé par "GOOS/GOARCH" et ${bin} par le chemin d'accès au binaire compilé.
    "*/*": ""
  },
  // Données utilisées pour remplir le manifest et les informations sur la version.
  "info": {
    // Nom de la compagnie. Défaut: [The project name]
    "companyName": "",
    // Nom du produit. Défaut: [The project name]
    "productName": "",
    // Version du produit. Défaut: '1.0.0'
    "productVersion": "",
    // Copyright du produit. Défaut: 'Copyright.........'
    "copyright": "",
    // Court commentaire sur le produit. Défaut: 'Built using Wails (https://wails.app)'
    "comments": ""
  },
  // 'multiple': Un installateur par architecture. 'single': Installateur universel unique pour toutes les architectures. Défaut: 'multiple'
  "nsisType": "",
  // Si l'application doit être masquée. Défaut: false
  "obfuscated": "",
  // Les arguments à passer à la commande garble lors de l'utilisation de l'option de masquage
  "garbleargs": ""
}
```

Ce fichier est lu par le CLI Wails lorsque vous exécutez `wails build` ou `wails dev`.

Les options `assetdir`, `reloaddirs`, `wailsjsdir`, `debounceMS`, `devserver` et `frontenddevserverurl` dans `wails build/dev` vont mettre à jour les configs du projet et deviendront les valeurs par défaut des exécutions suivantes.

Le Schema JSON pour ce fichier se trouve [ici](https://wails.io/schemas/config.v2.json).
